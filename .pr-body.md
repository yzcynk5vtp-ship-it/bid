### 4.2.1.5 下架和恢复 — 功能完善

#### 背景
现有下架/恢复基础功能已实现，但前端下架弹窗不符合蓝图要求。

#### 改动内容
| 文件 | 修改内容 |
|------|---------|
| `QualificationController.java` | 下架原因上限从 500 字符改为 **200 字符** |
| `RetireConfirmDialog.vue` | **新增** 下架确认弹窗：展示证书名称/编号 + 下架原因输入（≤200字符）+ "我已确认"勾选框 |
| `Qualification.vue` | 集成 `RetireConfirmDialog`，默认筛选排除 `RETIRED` |
| `QualificationControllerTest.java` | 新增 200 字符上限测试用例 |
| `RetireConfirmDialog.spec.js` | **新增** 前端弹窗测试（4 个用例） |

#### 需求核对
- [x] 下架原因必填，≤200 字符
- [x] "我已确认"勾选框必须勾选才能点击"确认下架"
- [x] 下架后状态变为"已下架"
- [x] 写操作日志（`@Auditable` 注解）
- [x] 列表默认不显示已下架（默认 statuses: IN_STOCK/EXPIRING/EXPIRED）
- [x] 恢复后状态由系统自动计算（`QualificationExpiryPolicy.evaluate()`）
