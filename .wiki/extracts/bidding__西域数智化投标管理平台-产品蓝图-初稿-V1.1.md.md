## **一、产品愿景**

### **1.1 平台定位**

数智化投标管理平台是面向西域投标全生命周期管理的智能化系统，致力于将标讯获取、项目立项、编标协同、知识复用、费用管理、结果闭环与管理分析统一到平台中，实现投标业务的数字化、智能化升级。

### **1.2 核心价值**

| 价值维度 | 描述 |
|-|-|
| **效率提升** | AI 自动拆解招标文件、智能提取关键信息，减少人工重复劳动 |
| **风险管控** | 全流程合规校验、废标风险预警、保证金闭环管控 |
| **知识沉淀** | 项目归档与资产沉淀体系，优秀方案可复用、可追溯，同时尽可能复用西域24年的行业经验 |
| **决策支持** | AI 商机预测、中标率分析、竞争情报洞察 |

### **1.3 目标用户**

| 角色 | 人员 | 核心职责 |
|-|-|-|
| 投标管理员 | Tina | 统筹投标全流程、管理标讯、分配任务、审核任务 |
| 投标组长 | Suki | 统筹投标全流程、管理标讯、分配任务、审核任务 |
| 投标专员 | Lily | 投标负责人，投标辅助人员，标书审核人，分配任务，审核任务 |
| 项目负责人 | Davaid | 标讯录入，标讯评估，项目立项 |

| 系统管理员 | Admin | 系统配置、权限管理、数据维护 |

**定位**：标讯中心是投标管理系统的**数据入口模块**，负责标讯信息的采集、管理和初步筛选，为后续的投标项目立项提供数据基础。

 **核心目标**

- 实现标讯多渠道采集（第三方平台、人工录入、批量导入）
- 实现标讯的智能去重和自动分配
- 实现标讯评估和投标决策的闭环流程
- 实现权限隔离，确保数据安全
- 所有操作留痕，支持审计追溯

 **业务范围**

- 标讯的自动采集与人工录入
- 标讯的分配、转派和跟踪
- 标讯评估表的填写和审核
- 标讯投标/放弃决策
- 标讯台账管理和导出
- 标讯源配置管理

#### 4.2.2 标讯状态与流转

**状态枚举**

| 枚举值 | 显示文本 | 标签颜色 | 说明 |
|-|-|-|-|
| PENDING_ASSIGNMENT | 待分配 | info（灰色） | 新拉取或新录入，尚未分配项目负责人 |
| TRACKING | 跟踪中 | primary（蓝色） | 已分配项目负责人，尚未提交评估 |
| EVALUATED | 已评估 | warning（橙色） | 项目负责人已填写并提交评估表，等待决策 |
| BIDDING | 投标中 | warning（橙色） | 已确认投标，已立项并进入投标流程 |
| WON | 已中标 | success（绿色） | 项目结果已确认中标，由项目模块回写 |
| LOST | 未中标 | danger（红色） | 项目结果已确认未中标，由项目模块回写 |
| ABANDONED | 已放弃 | info（灰色） | 投标管理员/组长/投标负责人选择放弃投标 |

- 投标中（BIDDING）与已评估（EVALUATED）同为 warning（橙色），表示均为"进行中/待处理"状态
- 已中标（WON）使用 success（绿色），表示正向结果

**状态流转规则**

| 流转 | 触发条件 | 操作人 | 说明 |
|-|-|-|-|
| 待分配 → 跟踪中 | 自动分配或手动分配 | 系统自动/投标管理员/投标组长 | CRM匹配成功自动分配；无匹配则需手动分配 |

#### 4.2.3 标讯创建

**数据来源**

| 来源方式 | 操作人 | 入口位置 | 说明 |
|-|-|-|-|
| 第三方平台自动拉取 | 系统自动/投标管理员 | 标讯源配置 + 一键获取按钮 | 定时轮询或手动触发 |
| CRM商机导入 | 系统自动 | 项目负责人在crm中立项，立项成功调用投标系统接口创建标讯 | 标讯相关字段从crm中传入 |
| 人工单条录入 | 所有角色（投标专员除外AI解析） | 列表页【人工录入】按钮 | 弹窗表单填写 |
| 批量导入（Excel） | 投标管理员/投标组长 | 列表页【批量导入】按钮 | 下载模板 → 填写 → 上传 |

**去重规则**

去重判断依据以下四个字段的组合**完全匹配**：

1. 项目名称（name）
2. 招标主体（purchaserName）
3. 报名截止时间（registrationDeadline）
4. 开标时间（bidOpeningTime）



**去重处理逻辑**

| 数据来源 | 重复判定时机 | 处理方式 |
|-|-|-|
| 第三方平台自动拉取 | 入库前 | 自动跳过，不导入，无提示 |
| 人工单条录入 | 点击保存时 | 弹窗提示，由操作人选择"覆盖"或"新建" |
| 批量导入（Excel） | 每行解析时 | 按去重规则筛选后直接导入 |
| crm商机 | 调用查询接口 | 已存在时关联该标讯（仅未立项状态） |

标讯列表页顶部操作区提供两个入口：

- 【标讯源配置】按钮：标讯源配置用于设置第三方平台自动拉取标讯的规则参数。仅**投标管理员**可操作，其他角色不展示该入口。
- 【一键获取标讯】按钮：手动触发标讯同步**配置页面字段明细操作按钮规则**测试连接按钮

  | 字段名称 | 字段类型 | 是否必填 | 默认值 | 关联条件/操作规则 | 校验&交互说明 |
  |-|-|-|-|-|-|
  | 标讯源平台 | 多选复选框 | 是 | 中国政府采购网（初始化时可根据实际需求） | 可选选项：中国政府采购网、各省招标网、第三方商机服务、企业招标平台 | 1. 至少必选1项，保存无选中则弹窗提示：请至少选择一个标讯源平台  <br/>2. 选中【第三方商机服务】→ 下方API端点、API密钥变为必填；未选中则API两项非必填、可置灰 |
  | API端点 | 单行文本输入框 | 条件必填 | 空 | 仅勾选「第三方商机服务」时必填 | 1. 校验URL格式（http/https） |
  | API密钥 | 密码隐藏输入框 | 条件必填 | 空 | 仅勾选「第三方商机服务」时必填 | 1. 纯文本录入，隐藏展示 |
  | 关键字 | 输入框+下拉选择 | 否 | 空 | 支持多关键字逗号分隔 | 为空则不做关键字过滤，全量拉取标讯。选项由客户初始化导入 |
  | 业务单位 | 输入框+下拉选择 | 否 | 空 | 支持输入或下拉选择 | 选项由客户初始化导入 |
  | 所属地区 | 下拉选择 | 否 | 全国（默认不限制） | 选项为全国+各省市 | 不选默认全国；选指定地区则只拉取对应地区标讯 |
  | 预算最小值(万元) | 数字输入框 | 否 | 0 | 整数 | 必须≤预算最大值，保存时校验，否则提示：预算最小值不能大于最大值 |
  | 预算最大值(万元) | 数字输入框 | 否 | 1000 | 整数 | 同上 |
  | 自动同步 | 开关按钮 | 否 | 关闭 | 开启/关闭二态 | 开启后后台按固定周期自动拉取标讯；关闭仅手动拉取，拉取频率初始化时根据西域实际需求在后台写入 |
  | 自动去重 | 复选框 | 否 | 勾选 | 可勾选/取消 | 取消即不做标题/编号去重，直接入库 |

  - **可用性**：仅勾选「第三方商机服务」且已填API端点+密钥时可点击，否则置灰不可点
  - **操作逻辑**：向后端传API地址+密钥，做连通性测试
  - **返回交互**：成功弹「连接测试成功」；失败弹「连接失败，请检查API端点和密钥」

### **4.3 投标项目**

投标项目管理模块是「西域数智化投标管理平台」的核心模块，贯穿从项目立项到结项复盘的全生命周期。通过数字化手段实现投标业务的标准化、可视化与智能化管理。

#### **4.3.1流程概览**

**流程说明：**

1、项目立项数据主要来自于【标讯中心】和【CRM系统】：

（1）标讯中心的标讯可通过三方平台推送或手工录入／导入。投标管理员／组长将其分配给项目负责人后，由负责人填写项目评估。评估通过后，进入项目立项。

（2）CRM系统触发投标时，会查询投标系统中是否存在对应标讯：如存在，则自动关联（项目详情页面有标讯字段关联）；如不存在，则手动录入标讯并关联，然后进入项目待立项阶段（项目评估信息自动带入）。

**触发说明：**

- 前置条件：

  - 项目状态必须为 `CLOSED`
  - 项目下至少有一份「标书文件」分类的文件
  - 项目下至少有一项评分项
- 处理逻辑（异步任务模型）

| 步骤 | 动作 |
|-|-|
| 1 | 项目结项后→ 系统自动创建一个**异步任务** |
| 2 | 后台任务取标书文件（输入源 A）+ 评分项（输入源 B） |
| 3 | 调用 AI执行任务：把标书内容按每个评分项**分类切片**，**只抽取不改写** |
| 4 | AI 输出：评分项原文 → 标书对应应答内容片段 的配对集 |
| 5 | 每个配对写入案例库（业务字段：评分项原文 / 应答片段 / 来源项目 / 中标结果 / 行业 / 客户类型 / 产品线 / 创建时间） |
| 6 | 任务结束 → 通过**消息中心**向触发管理员推送通知：成功/失败 + 入库数量 + 跳转链接 |

**异常处理**

| 场景 | 系统处理 | 兜底 |
|-|-|-|
| 标书文件缺失 | 前置条件不满足，按钮 disabled | 上传标书后即可触发 |

## **六、非功能性保障**

### **6.1 性能需求**

| 能力 | 响应时间要求 |
|-|-|
| 招标文件拆解 | 单文档 ≤ 30s（PDF ≤ 50页） |
| 项目评分 | 单次评估 ≤ 5s |
| 标书初审 | 单文档 ≤ 60s |
| 历史方案提取 | 单项目 ≤ 120s |
| 商机预测 | 单客户 ≤ 3s |
| 页面加载 | ≤ 2s |

### **6.2 安全需求**

- AI 分析结果仅授权角色可查看
- 涉及客户敏感数据的预测结果需脱敏展示
- AI 调用日志完整记录，不可删除
- 涉及商业机密的标书内容传输加密
- 权限隔离，仅授权角色可查看操作

### **6.3 容错需求**

| 异常场景 | 处理策略 |
|-|-|
| AI 服务不可用 | 降级提示"AI 服务暂时不可用"，保留人工操作入口 |
| 分析超时 | 自动保存当前进度，支持手动重试 |
| 内容识别失败 | 标记识别失败区域，提示用户人工确认 |
| 流程中断 | 自动保存当前进度，重启后接续执行 |
