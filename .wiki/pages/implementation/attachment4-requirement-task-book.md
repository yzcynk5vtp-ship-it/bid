# 附件4需求任务书交付基线

本页用于把《附件4：西域数智化投标管理平台建设项目需求任务书》沉淀为项目交付核查基线。后续范围评审、开发排期、UAT、上线、试运行和终验，均需与 [[requirements]]、[[contract-constraints]]、[[implementation/sow-2026-v1-4]] 联合核对。

## 1. 文件定位

| 项 | 口径 |
|---|---|
| 文件性质 | 合同附件中的需求任务书，是功能、非功能、实施服务、交付文档、验收和质保要求的重要事实源 |
| 当前 wiki 状态 | 已进入 `.wiki/sources/contract/`，并完成抽取 |
| 执行关系 | 主合同和 SOW V1.4 优先；附件4用于校验需求完整性、交付物完整性和验收条件 |
| 冲突处理 | 与主合同、SOW、蓝图确认件或双方书面确认件不一致时，必须形成书面澄清，不应在开发中自行裁剪 |

## 2. 建设目标

| 目标 | 必须覆盖的交付含义 |
|---|---|
| 全生命周期数字化管控 | 从商机/标讯、评估、立项、任务、标书、评审、封装提交到结果归档形成线上闭环；支持关键节点监控、预警、督办和过程看板 |
| 提升投标质量与中标率 | 将 SOP、质量控制点、模板、资质、案例、竞对信息和标书辅助能力沉淀到系统，降低个人经验依赖和人工丢标风险 |
| 企业级投标数据中心 | 统一采集项目信息、投入成本、结果数据，支撑中标率、投入产出、竞对动态、报价策略和复盘分析 |
| 开放集成与灵活扩展 | 支持 CRM、财务、企业微信、第三方标讯/商机服务等集成，并具备配置化、扩展性和长期演进能力（OA 集成已取消） |

## 3. 采购范围

| 范围 | 核查点 |
|---|---|
| 软件系统永久使用权 | 本地化部署版本，授权范围、用户数和模块范围需回到合同正文与报价清单确认 |
| 实施方案 | 包含系统部署、数据初始化、用户培训、业务流程梳理、系统配置和既有系统集成服务 |
| 售后服务 | 包含维护、故障排查、技术支持、升级服务和质保期内缺陷处理 |

## 4. 核心业务能力

| 能力域 | 需求任务书要求 | Wiki 对应 |
|---|---|---|
| 投标流程线上化与自动化 | 支持投标项目从商机/标讯获取、标前评估、项目立项、任务分工、文件编制、内部评审、中标反馈到归档的生命周期管理；支持项目商机联动创建投标项目 | [[requirements]]、[[business-process]]、[[modules]] |
| 流程引擎与预警 | 可自定义流程引擎；覆盖标讯录入追踪、立项、任务、编制、评审、中标反馈；基于存量标讯、合同到期和各阶段关键节点自动提醒 | [[business-process]]、[[deployment]] |
| 核心资源集中治理 | 建立资质、业绩、人员证书、标书模板、历史项目文件、经验知识、外部招标平台账户等资源库，并支持检索复用和借阅账户管理 | [[modules]] |
| 标书辅助 | 支持标书模板管理、关键信息自动检查、辅助书写、资质匹配、商务条款符合性和合规检查点 | [[ai-capabilities]]、[[requirements]] |
| 协同与风控 | 支持项目团队接收任务、督促工作，跟进保证金、费用、投标过程记录、竞争对手信息和项目结果闭环 | [[business-process]]、[[modules]] |
| 中标结果沉淀 | 记录中标 SKU 数量、品类、折扣、账期等信息，归档历史项目数据，辅助市场分析 | [[business-process]]、[[data-model]] |
| 数据分析决策 | 提供仪表板和多维报表，从项目、区域、中标率、资源投入、客户、竞对等维度呈现业务全景 | [[requirements]]、[[modules]] |
| 企业级集成与权限 | 支持 OA、CRM、财务、第三方商机/标讯服务、企业微信组织架构等安全可靠交换；按组织架构、项目组、用户配置系统和数据权限 | [[architecture]]、[[roles-and-permissions]] |

## 5. 技术与非功能基线

| 类别 | 附件4要求 | 交付核查口径 |
|---|---|---|
| 部署模式 | 支持甲方指定私有云部署，并对接既有基础设施、网络和安全体系 | 生产部署方案、环境清单、网络白名单、备份恢复和上线切换方案需可验证 |
| 架构与浏览器 | B/S 架构，支持 Chrome、Edge、Firefox，不依赖插件或本地客户端 | 前端兼容性和无插件访问是验收前置项 |
| 安全 | HTTPS、细粒度权限、全流程操作日志、等保相关要求 | 安全方案、权限矩阵、日志审计、账号密码策略、备份灾备方案需纳入验收证据 |
| 集成性 | 标准 API，具备与 OA、CRM、企业微信集成能力 | 接口设计文档、联调记录、接口日志、异常报警和接口开发说明需闭环 |
| 性能 | 关键业务并发快速响应；普通交互需满足体验要求；具备性能测试、监控和调优方法 | 性能测试报告、监控方案和容量扩展说明需在上线前确认 |
| 可用性 | 工作时间正常运行时间大于 99.9%，非工作时间大于 99.8%；合同/SOW另有 99.5% 年可用性口径 | 蓝图或 SOW 阶段必须书面确认最终适用指标 |
| 移动支持 | 支持微信、企业微信小程序或移动浏览器进行关键审批、进度查看和预警接收 | 本期范围需与 SOW 白名单和蓝图确认件核对 |
| 扩展性 | 支持流程、页面、表单、报表、BI 图表、数据表配置；报表可由甲方定制；提供二次开发接口、工具和平台 | 超出标准能力的低代码、报表自定义、数据库直连等需在蓝图阶段明确边界 |
| 可维护性 | 系统易维护、易升级、易监控、易故障隔离，需说明运维人力、物力和技术支持体系 | 运维管理制度、监控日志、应急预案和人员配置需作为交付物 |

## 6. 数据集成细则

| 要求 | 核查点 |
|---|---|
| 主数据映射 | 需支持外部系统编码、名称差异的配置映射，并按平台编码转换汇总 |
| 自动/手动取数 | 接口执行方式可选择自动同步或手动同步 |
| 数据源适配 | 支持主流数据库或数据仓库对接，如 MySQL、PostgreSQL、ClickHouse、Hive 等 |
| 第三方系统 | 支持 OA、北森、帆软等主流信息化系统对接，最终以蓝图确认接口清单为准 |
| 异常治理 | 接口数据自动校验，异常自动报警，接口状态可监控且有详细接口日志 |
| Office 集成 | 支持与 MS Office/WPS 集成，系统可生成 Word、PDF、Excel 报告 |
| 调度与消息 | 接口可通过调度、计划任务或人工干预执行；支持监听和消费 Kafka 等中间件信息 |
| 统一认证授权 | 支持与其他应用统一认证、统一授权 |

## 7. 安全与权限细则

| 类别 | 核查点 |
|---|---|
| 安全体系 | 方案需覆盖网络安全、数据安全、保密性、系统安全、备份恢复和灾备应急 |
| 密码策略 | 登录必须使用有效用户名和口令，并具备完善密码策略 |
| 权限模型 | 支持用户、用户组、角色等多级权限，支持功能权限、数据权限和统一权限管理 |
| 数据安全 | 允许授权用户访问合法数据；禁止越权操作；支持数据库级、表级、字段级数据安全控制 |
| 操作留痕 | 用户设置、变更、密码修改、功能查询修改、导入导出、接口调用、异常和管理员操作均需留痕 |
| 等保要求 | 附件4提到系统安全等级 3 级或以上安全方案；与合同/SOW最终等级口径需书面确认 |
| 防护项 | 方案需覆盖防病毒、防篡改、防 DDoS、防 SQL 注入、防 DNS 攻击、上传下载权限限制、可疑 IP 跟踪等 |

| 阶段 | 关键活动 | 交付口径 |
|---|---|---|
| 项目准备 | 成立项目小组、启动会准备、实施主计划、演示环境安装 | 项目组织成员清单、实施主计划、启动会 PPT、演示环境 |
| 蓝图设计及系统配置 | 平台演示、现状了解、差异收集、蓝图规划、解决方案设计 | 调研流程清单、需求适配分析、配套需求确认、接口/流程/原型设计、业务落地方案、蓝图确认 |
| 系统实现 | 功能设计、程序实现、功能测试、基础数据收集、集成测试、最终用户培训、操作手册 | 二次开发说明、数据结构说明、源代码、配置报告、TASKC 任务和 BUG 清单、测试用例、培训资料、操作手册 |
| 上线准备 | 上线策略、服务器和 IT 流程准备、系统移植、正式环境建立、静态/动态数据初始化 | 上线切换方案、系统安装手册、部署手册、数据初始化记录、上线确认函 |
| 上线支持 | 使用者支持验证、问题处理、结案、上线首月运行评估 | 上线运行问题跟进表、验收报告、项目验收测试报告、项目总结 |

## 9. 项目文档交付清单

| 阶段 | 必备交付物 |
|---|---|
| 项目准备 | 项目组织成员清单、投标管理系统项目实施主计划、启动会 PPT |
| 蓝图设计及系统配置 | 调研流程清单、项目需求适配分析报告、配套需求适配确认书、接口设计文档、流程设计文档、原型设计文档、项目业务落地解决方案、蓝图阶段问题清单、蓝图汇报 PPT、蓝图确认函 |
| 系统实现 | TASKC 任务清单、TASKC BUG 修复清单、项目实施配置报告、数据结构说明文档、二次开发功能说明文档、开发程序源代码、系统上线配置手册 |
| 上线准备 | 测试用例及操作截图、测试报告、部署手册、项目培训资料和教材、系统功能操作手册、业务功能操作手册、项目运维管理制度、测试问题清单 |
| 上线支持 | 项目验收测试报告、验收报告、上线切换方案、运行跟踪报告、系统安装手册、系统项目上线运行问题跟进表、投标管理系统项目总结、上线确认函 |

所有文档需按双方确认的工作计划提交；除源代码外，文档应以中文提供；电子版和纸质版均需满足要求。关键节点或关键文档需双方盖章、签字确认后方可进入下一阶段。

## 10. 人员配置基线

| 要求 | 核查点 |
|---|---|
| 项目经理 | 现场全职项目经理，负责需求适配和实施开发两部分服务；5 年以上投标管理系统项目管理经验；不得由项目总监兼任 |
| 现场出席 | 项目经理在需求调研和方案设计阶段全程出席项目现场；方案设计负责人在配置实施阶段每周至少现场 2 天 |
| 顾问与骨干 | 必须有专门顾问负责需求适配；每个模块必须有骨干顾问负责技术支持 |
| 人员稳定 | 经甲方面试合格并进入项目组后原则上不得变更；离职等特殊情况需提供同级别人员并经甲方认可 |
| 否决风险 | 人员配置达不到附件4要求，甲方有权作为投标中标否定条款；合同执行阶段也会影响付款和验收 |

## 11. 进度与质量控制

| 项 | 附件4口径 |
|---|---|
| 进度原则 | 总体规划、分阶段实施；当前统一以 2026-07-10 正式上线为执行基线 |
| 执行澄清 | 当前执行应以主合同、SOW V1.4、2026-04-30 本周客户确认的 2026-07-10 正式上线时间和双方签发计划为准；如需阶段性核心功能可用，需书面拆分阶段可用范围与 2026-07-10 正式上线门禁 |
| 蓝图质量控制 | 评审需求适配资料，至少覆盖整体需求、分期边界、流程图、表单、权限角色、数据标准化规范 |
| 方案质量控制 | 评审流程设计文档和业务落地解决方案，关注合理性、便捷性和高效性 |
| 实施质量控制 | 评审配置实施、安装调试、系统测试资料，包括操作手册、测试报告、切换上线方案等 |
| 上线质量控制 | 评审培训推广、上线准备、运行和验收资料，包括培训教材、业务/系统手册、运行报告、验收报告等 |
| 通过标志 | 质量控制点评审通过，并由甲方项目经理及质量控制小组联合签字 |

## 12. 验收、质保与保密

| 类别 | 要求 |
|---|---|
| 软件平台验收 | 平台及模块满足全部功能需求，符合数字化建设、信息化总体规划、等保要求，并具备接口扩展性 |
| 文档资料验收 | 纸质与电子文件满足需求任务书要求，重要文件通过甲方组织评审 |
| 培训验收 | 培训计划和验收标准经双方确认，甲方人员能基本掌握操作并完成知识转移 |
| 实施过程验收 | 通过全部质量控制点评审 |
| 系统整体验收 | 试运行验证后，系统符合需求任务书全部要求并达到建设目标 |
| 质保服务 | 系统完成试运行并通过最终验收之日起 12 个月；每三个月一次系统安全检测；保修期内按需免费二次培训 |
| 技术支持 | 免费技术支持包括应用修改、功能增加、系统维护、技术培训和系统升级 |
| 知识产权 | 业务数据、过程文档、二次开发部分知识产权归甲方；定制开发源代码和设计文档需完整交付 |
| 保密 | 甲方资料、数据、软件、程序只能用于本项目技术支持和服务，不得扩散或另作他用 |
