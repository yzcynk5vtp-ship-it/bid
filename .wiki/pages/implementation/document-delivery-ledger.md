# 文档交付台账

本台账用于跟踪附件4、附件6、主合同和 SOW V1.4 要求的正式项目文档。附件4提供交付和验收框架，附件6提供功能点颗粒度；附件6优先使用人工结构化 Markdown，Excel 原件保留为原始来源。每份文档必须同时满足“电子版归档、必要时纸质版提交、甲方确认、关键节点签字/盖章”四类要求，才能作为阶段验收和付款证据。

## 状态口径

| 状态 | 含义 |
|---|---|
| 待编制 | 尚未形成正式文档 |
| 编制中 | 已有草稿或材料，尚未提交评审 |
| 待评审 | 已提交甲方或内部评审，等待意见 |
| 待签字 | 内容已通过，等待双方签字/盖章 |
| 已签发 | 已完成签字/盖章，可作为验收或付款证据 |
| 需更新 | 因范围、计划、问题或客户意见变化，需要补版 |

## 台账字段

| 字段 | 填写规则 |
|---|---|
| 编号 | `DD-阶段-序号`，其中 `DD` 表示 Deliverable Document（交付文档），用于会议纪要、周报和验收报告引用 |
| 阶段 | 项目准备、蓝图设计及系统配置、系统实现、上线准备、上线支持、试运行/终验 |
| 交付文档 | 使用附件4或合同中的正式名称 |
| 验收用途 | 说明该文档对应的阶段门禁、付款节点或验收类别 |
| 责任角色 | 默认乙方项目经理统筹，按内容指定实施、研发、测试、运维或业务负责人 |
| 当前状态 | 使用本页状态口径 |
| 计划提交 | 以 [[team-and-timeline]] 和双方确认计划为准，变更需记录 |
| 证据路径 | 填正式文档归档路径、签字扫描件路径、会议纪要或报告链接 |
| 签字要求 | 甲方确认、双方签字、双方盖章、评审确认、按需归档 |

## 更新规则

1. 项目经理每周例会前更新“当前状态、计划提交、证据路径、备注风险”。
2. 所有进入“待签字/已签发”的文档，必须在周报中列明文档版本号、提交日期、确认人和签字状态。
3. 任一文档计划延期超过 3 个工作日，需进入 [[implementation/risk-register]] 并给出纠偏动作。
4. 已签发文档不得直接覆盖；如需修改，应新增版本并保留旧版签发记录。
5. 终验前，本页所有附件4正式文档应达到“已签发”或有双方书面豁免。
