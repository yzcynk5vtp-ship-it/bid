# Specification: 动态工作台配置体系与数据域权限

## 1. 业务目标
将基于预设角色的静态工作台（硬编码组件展示）升级为符合 B2B 系统标准的企业级“动态配置化工作台”。解决“同一张卡片，不同人看不同数据”以及“同一岗位，因权限不同看到不同组件”的灵活定制需求。

## 2. 核心方案 (分三阶段)

### 阶段 1：功能权限解耦（视图级拦截）
*   **前端修改**：`Workbench.vue` 放弃基于 `role` 的死板判断，改为基于 `permissions`（菜单/组件权限点）进行 `v-if` 控制。
*   **依赖条件**：后端需在登录/会话恢复时下发准确的用户权限数组（如 `dashboard:component:tender_list`）。

### 阶段 2：数据域权限（数据级拦截）
*   **后端修改**：在查询项目、标讯、审批等列表的 Controller/Service 层，接入 Data Scope（数据范围）拦截器。
*   **实现机制**：通过 AOP/MyBatis 拦截器，根据用户的 `dataScope` 属性（本人、本部门、全公司等），自动在底层 SQL 中注入 `WHERE` 条件。前端不负责核心业务数据的越权过滤。

### 阶段 3：JSON 配置驱动渲染（远期目标）
*   **前后端配合**：后端提供配置存储和查询接口，返回组件编排 JSON 协议。前端通过 `<component :is="..">` 动态渲染工作台布局。取代目前的“全量引入 + 条件隐藏”模式。
