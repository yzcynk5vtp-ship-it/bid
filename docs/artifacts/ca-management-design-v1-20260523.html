资源管理 §4.5 蓝图功能实现方案 —— 将分散的 CA/证书管理功能整合为统一管理看板

概览

CA（Certificate Authority）证书是投标过程中重要的数字身份凭证，用于标书电子签章。 目前 CA/证书管理功能分散在 businessqualification 和 platform 两个后端模块， 前端没有统一的看板页面。蓝图 §4.5 要求一个集中式管理页面。

🔑
CA 证书
实体 CA + 电子 CA 统一管理
🔄
借用归还
全生命周期跟踪
⏰
到期预警
有效期监控 + 智能提醒

现状分析

模块 已有功能 缺失部分
businessqualification 证书 CRUD、借阅归还、过期提醒、AI 上传解析 无统一看板，无 CA 类型区分（实体/电子）
platform 平台账号管理、密码加密、账号借用 CA/印章管理仅做基础登记，无可视化看板
resources/BarCertificate BAR 证书 CRUD、借用归还 仅限于 BAR 资产，不覆盖全部 CA 类型

核心问题

CA 数据分散在三个模块中，用户需要分别进入不同页面管理。缺失：统一看板、到期可视化、CA 类型分类展示、一键借用。

功能规格

📊 统计卡片区

  • • 总证书数：CA 证书的总数量
  • • 即将到期：90 天内到期的证书数（黄色）
  • • 已过期：超过有效期的证书数（红色）
  • • 借出中：当前被借用的证书数
  • • 数据来源：GET /api/knowledge/qualifications/overview

🔍 筛选与搜索

  • • CA 类型：实体 CA / 电子 CA / 全部
  • • 状态：正常 / 即将到期 / 已过期 / 借出中
  • • 关键词搜索：名称、编号、保管人
  • • 全部前端过滤，无需后端接口

📋 证书列表

  • • 列：证书名称、类型（实体/电子）、印章类型、保管人、有效期、状态标签
  • • 状态标签颜色：绿色(正常) / 黄色(即将到期) / 红色(已过期) / 灰色(借出中)
  • • 点击行 → 打开详情抽屉
  • • 排序：支持按有效期、名称排序

📂 详情抽屉

  • • 证书基本信息：名称、编号、发证机关、持有人、有效期
  • • CA 类型字段：实体/电子、印章类型、绑定平台
  • • 借用历史记录列表
  • • 操作按钮：编辑、删除、借用、归还
  • • 附件下载（如有）

➕ 新增/编辑弹窗

  • • 表单字段：名称、证书编号、发证机关、持有人、CA 类型、印章类型、有效期、关联平台
  • • 校验：名称必填、有效期不能早于今天、证书编号唯一
  • • 保存后自动刷新列表
