# 执行计划目录 (docs/exec-plans/)

复杂任务的**执行计划**作为代码库交付物落地于此。计划是"一等公民"，不只在聊天窗口里管理——后续 AI 运行或协作 agent 可在不依赖外部上下文的情况下接手工作。

> 本目录是模板对齐目录，与既有 `docs/plans/`、`docs/archives/`、`conductor/` **并存**，不搬迁现有文件。

## 目录结构

| 目录/文件 | 职责 | 维护频率 |
|---|---|---|
| `active/` | 正在执行的复杂任务进度与决策日志 | 高 |
| `completed/` | 已完成任务归档入口（指向 `docs/archives/`） | 低 |
| `tech-debt-tracker.md` | 技术债追踪器，供后台扫描重构 | 中 |

## 与现有计划目录的映射

| 概念 | 本目录 | 现有位置（保持不动） |
|---|---|---|
| 活跃开发计划 | `active/` | `docs/plans/`（活跃 PRD/设计稿） |
| 已完成计划归档 | `completed/` | `docs/archives/plans-2026-{03,04,05}/`（历史） |
| 任务编排 | — | `conductor/tracks/`（编排 track） |

## 落计划约定

- **小型修改**：使用轻量级临时计划即可，不必在此建文件。
- **复杂任务**：在 `active/` 下建 `<task-slug>-plan.md`，包含：
  - 目标与背景
  - 进度清单（勾选项）
  - 决策日志（含未采纳方案与原因）
- 任务完成后，将计划移入 `completed/` 或合并到 `docs/archives/`。
