## 一、项目级子身份说明

| 身份 | 角色 | 定义 |
|---|---|---|
| 投标负责人 | 投标专员 | 由投标管理员在立项时指派，主导标书制作、任务分配、投标提交 |
| 投标辅助人员 | 投标专员 | 与投标负责人权限一致，协办标书制作 |
| 标书审核人 | 投标专员 | **临时选定**（非立项时预指派），投标负责人上传最终标书后选择，必须是**未参与本项目的人** |
| 执行人 | 任意角色 | 被分配具体任务的人（含项目负责人），仅对自己被分配的任务有操作权 |

> **注**：同一投标专员在项目 A 可能是投标负责人，在项目 B 可能是标书审核人，权限随项目身份变化。

## 五、关键业务规则汇总

1. **立项即锁定**：审核通过后项目基本信息不可编辑。
2. **投标团队二元制**：负责人+辅助人权限一致，共同管理标书制作。
3. **标书审核人临时选定**：不在立项时预指派；投标负责人上传标书后临时选择未参与本项目的人审核。
4. **任务审核链**：执行人提交 → 负责人/辅助人审核（通过/驳回）。
5. **标书审核链**：负责人/辅助人上传标书 → 标书审核人审核 → 负责人/管理员/组长/辅助人提交投标。
6. **文档双层权限**：全体参与人可上传；仅管理员/组长/负责人/辅助人可查看下载；仅管理员可删除。
7. **结果/复盘无需审核**：提交即生效。
8. **保证金双向**：项目负责人填写 → 管理员/组长/负责人/辅助人审核。
9. **结项四人可触发**：管理员/组长/投标负责人/投标辅助均可发起结项和审核。
10. **二次招标受限**：管理员/组长/投标负责人/投标辅助可触发，跳转回标讯模块。
