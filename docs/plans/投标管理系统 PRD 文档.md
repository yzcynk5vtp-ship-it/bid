## 2\. 核心业务流程（最终版）

```Plain Text
项目立项 → 标书编制中 → 评标中 → 结果确认 → 项目复盘 → 项目结项
```

### 2\.1 流程说明

1. **项目立项**
销售基于现有标讯转为投标项目，填写完整项目基础信息，提交进入下一阶段。

2. **标书编制中**
投标管理员分配主 / 副投标负责人 → 系统自动拆任务 → 指派执行人 → 任务审核完成。

3. **评标中**
标书提交后进入评标跟踪，支持手动切换子状态。

4. **结果确认**
投标负责人登记中标 / 未中标 / 流标 / 弃标并上传凭证。

5. **项目复盘**
按结果类型填写复盘内容，审核通过进入结项。

6. **项目结项**
展示保证金信息 → 登记是否退回 → **必须已退回才能提交结项** → 提交后项目锁定归档。

## 3\. 项目阶段详细需求（开发核心）

### 3\.1 阶段 1：项目立项

**触发条件**：销售从标讯点击「转为投标项目」
**操作角色**：销售 / 业务负责人

#### 3\.1\.1 必填写字段

|字段|类型|必填|规则|
|---|---|---|---|
|业主单位|文本 / CRM 查询|是|标讯自动带出|
|入围家数|数字|是|\-|
|合同期限|日期范围|是|\-|
|项目类型|下拉|是|公开招标 / 邀标 / 其他|
|客户类型|下拉|是|央企 / 地方国企 / 其他|
|营业收入|文本|是|\-|
|开标时间|日期时间|是|提交后锁定|
|投标月份|自动 / 下拉|是|取自开标时间|
|业务负责人|人员选择|是|默认当前用户|
|归属部门|部门|是|自动关联，快照保存|
|竞争对手|多选文本|否|可编辑|

#### 3\.1\.2 约束

- 信息不完整无法提交

- 开标时间、业主单位提交后**不可随意修改**

---

### 3\.2 阶段 2：标书编制中

**触发条件**：立项完成
**操作角色**：投标管理员、投标负责人、任务执行人

#### 3\.2\.1 功能点

1. 分配 **主投标负责人、副投标负责人**

2. 系统根据招标文件**自动拆解任务**

3. 支持手动增删改任务

4. 任务指派负责人、设置截止时间

5. 任务状态：

    - 待办 → 待审核 → 已完成

6. 审核不通过可退回并填写原因（必填留痕）

7. **全部任务完成后方可进入下一阶段**

#### 3\.2\.2 任务状态规则

|状态|操作人|动作|
|---|---|---|
|待办|执行人|开始处理、上传交付物|
|待审核|执行人|提交审核|
|已完成|投标负责人|审核通过|
|退回待办|投标负责人|驳回并留言|

---

### 3\.3 阶段 3：评标中

**触发条件**：标书编制全部完成
**操作角色**：投标负责人

#### 3\.3\.1 子状态（手动切换）

1. 评标中

2. 评标结果已出，待上会

3. 评标结果公示

#### 3\.3\.2 功能

- 展示项目关键信息

- 上传评标过程凭证

- 无需审批，仅做进度跟踪

---

### 3\.4 阶段 4：结果确认

**触发条件**：评标结果公示
**操作角色**：投标负责人

#### 3\.4\.1 结果类型（必选）

- 中标

- 未中标

- 流标

- 弃标

#### 3\.4\.2 约束

- 必须上传对应凭证（中标通知书 / 公告 / 说明等）

- 提交后自动进入 **项目复盘**

---

### 3\.5 阶段 5：项目复盘

**触发条件**：结果确认完成
**操作角色**：投标负责人、投标管理员

#### 3\.5\.1 复盘必填内容

|结果|复盘字段|
|---|---|
|中标|中标优势、亮点、建议|
|未中标|丢标原因、问题、改进措施|
|流标|流标原因、应对方案|
|弃标|弃标原因、决策说明|

#### 3\.5\.2 规则

- 所有项目**必须复盘**

- 复盘提交后需管理员审核

- 审核通过 → 进入 **项目结项**

---

### 3\.6 阶段 6：项目结项（含保证金・最终阶段）

**触发条件**：复盘审核通过
**操作角色**：业务负责人、投标负责人、投标管理员

#### 3\.6\.1 结构说明

**保证金管理作为项目结项的子模块**，不独立成阶段。

#### 3\.6\.2 结项页面字段

|字段|来源|必填|规则|
|---|---|---|---|
|保证金额|立项自动带出|是|不可编辑|
|缴纳方式|立项自动带出|是|不可编辑|
|是否退回|下拉：是 / 否|是|\-|
|退回日期|日期选择|是|仅 “是” 时显示|
|退回凭证|上传|是|仅 “是” 时显示|
|结项说明|文本|否|\-|

#### 3\.6\.3 前端强校验（开发必须实现）

```Plain Text
if (是否退回 == 否) {
   【提交结项】按钮 = 禁用
   提示：请完成保证金退回后再提交结项
}
```

#### 3\.6\.4 结项成功后

- 项目全字段**锁定不可修改**

- 所有资料自动归档

- 状态变为：已结项

## 5\. 通用功能与约束

- 单文件 ≤50MB

- 归档后可批量下载

### 5\.3 日志留痕

### 5\.4 阶段流转规则

**不允许跨阶段跳转**，必须按顺序：

```Plain Text
立项 → 标书编制 → 评标中 → 结果 → 复盘 → 结项
```
