## 一、角色定义及项目级子身份说明

### 角色（role code）

| 角色 | code | 核心职责 |
|---|---|---|
| 投标系统管理员 | bid-SystemAdmin | 拥有一切权限 |
| 投标管理员 | bidAdmin | 统筹项目阶段全流程、各阶段审核任务，任务分配、投标提交 |
| 投标组长 | bid-TeamLeader | 同投标管理员（当前权限一致，角色独立预留未来分化） |
| 投标项目负责人 | bid-projectLeader | 提交项目立项，标准制作阶段任务执行人，项目结项提交人 |
| 投标专员 | bid-Team | 投标负责人 / 投标辅助人员 / 标书审核人 |
| 跨部门协同人员 | bid-otherDept | 标准制作阶段任务执行人，标书审核人 |
| 行政人员 | bid-administration | 资质证书维护（不涉及标讯模块） |

### 项目级子身份（随项目变化）

| 项目级子身份 | 角色 | 定义 |
|---|---|---|
| 投标负责人 | 投标专员 | 由投标管理员在立项时指派，主导标书制作、任务分配、投标提交 |
| 投标辅助人员 | 投标专员 | 与投标负责人权限一致，协办标书制作 |
| 标书审核人 | 投标专员/跨部门协同人员 | **临时选定**（非立项时预指派），投标负责人上传最终标书后选择 |
| 任务执行人 | 任意角色 | 被分配具体任务的人（含投标项目负责人），仅对自己被分配的任务有操作权 |

> 同一投标专员在项目 A 可能是投标负责人，在项目 B 可能是标书审核人，权限随项目身份变化。

### 符号说明
- ✅ 有权限
- — 不涉及
- 全量 数据范围=所有项目
- 自己的 自己负责的项目
- 参与的 自己参与的项目（任一身份）

## 五、关键业务规则汇总

1. **立项即锁定**：审核通过后项目基本信息不可编辑。
2. **投标团队二元制**：投标负责人+投标辅助人权限一致，共同管理标书制作。
3. **标书审核人临时选定**：不在立项时预指派；投标负责人上传标书后临时选择跨部门人员审核。
4. **任务审核链**：任务执行人提交 → 投标负责人/投标辅助人员审核（通过/驳回）。
5. **标书审核链**：投标负责人/投标辅助人员上传标书 → 标书审核人审核 → 投标管理员/投标组长/投标负责人/投标辅助人员勾选已完成投标复选框，点击提交投标。
6. **文档双层权限**：全体参与人可上传；仅 投标管理员/投标组长/投标负责人/投标辅助人员 可查看下载；仅 投标管理员 可删除。
7. **结果/复盘无需审核**：提交即转至下一阶段。
8. **保证金双向**：投标项目负责人填写 → 投标管理员/投标组长/投标负责人/投标辅助人员审核。
9. **项目结项由投标项目负责人可触发**：投标管理员/投标组长/投标负责人/投标辅助人员可审核。
10. **二次招标受限**：投标管理员/投标组长/投标负责人/投标辅助人员/投标项目负责人 可触发，跳转回标讯模块。
