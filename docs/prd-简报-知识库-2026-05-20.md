## 一、 需求大纲与核心功能清单

### 1. 方案管理 (项目档案与案例库)
*   **项目档案 Tab (项目级文件归档台账)**
    *   **档案自动创建**：立项后自动建立档案，结项时不触发额外动作，过程中上传的文件即时打标签归档。
    *   **档案台账列表**：每行代表一个项目（而非文件），支持项目名称、文档分类多选、负责人、项目类型/状态等检索。
    *   **双轴时间筛选**：支持“上传时间”与“结项时间”双时间轴独立进行区间筛选。
    *   **气泡悬停明细 (Popover)**：列表“归档文件数”默认显示总数（如 `156 份`），鼠标悬停 `0.5s` 后弹出气泡分类展示明细（招标文件 3/标书文件 5/...）。
    *   **详情抽屉 (Drawer)**：行点击触发从右向左滑出的抽屉（默认宽度 60%，约 900px），包含基础信息模块（只读）、文件清单（支持单文件下载与 PDF 预览，上传时间倒序）、操作日志（以时间轴时间倒序呈现预览、下载、导出明细）。
    *   **台账与文件包导出**：
        *   **📊 导出台账**：Excel 文件，双表头（项目基本信息、文件清单）。
        *   **📦 导出文件包**：ZIP 压缩包，目录格式为 `[项目名]/[文档分类]/原文件名`，包内含 `_台账.xlsx` 描述汇总。
*   **案例库 Tab (AI 驱动的案例沉淀与复用)**
    *   **AI 案例沉淀 (异步任务)**：当项目进入结项状态 (`CLOSED`)，系统自动拉取「标书文件」与「评分项」，异步调用大模型进行内容分类切片（只抽取不改写），并将“评分项原文 -> 标书应答片段”自动存入案例库，完成后推送站内信/企业微信通知。
    *   **案例平铺网格卡片**：案例库默认使用平铺卡片（而非表格），支持模糊检索、多轴日期筛选、评分项类别下拉。
    *   **一键复用 (📋 复用)**：点击卡片或详情底部的复用按钮，直接将应答片段原文复制到系统剪贴板并提示 Toast，同时案例的 `reuseCount + 1`，保证低扰动。
    *   **案例详情抽屉**：固定宽度 560px，从右滑入，支持展示“招标要求原文”、“AI应答片段全文”、“关联元信息（项目、中标状态、客户类型等）”、“相似案例推荐（点击在当前抽屉替换）”以及“复用记录”。
    *   **案例下架**：管理员支持点击“下架”（软删除），确认后从前台案例库中隐藏。

### 2. 资质证书 (资质与信用资产)
*   **集中管理与录入**：支持证书名称、等级、认证机构、唯一证书编号、发证日期、有效期（必须晚于发证日期）、代理人及联系方式、认证范围、附件上传（支持单个 PDF/JPG/PNG，最大 10MB）。
*   **AI 智能识别回填**：上传资质文件时，AI 自动解析核心信息（发证日期、有效期、证书编号、认证范围等）并回填至表单对应输入框。
*   **证书到期提醒**：到期前 3 个月开始，通过站内信 + 企业微信持续通知行政与投标管理员。
*   **资质安全借阅**：借阅时必须先提交 OA 或系统内置审批单（`QUALIFICATION_BORROW`），审批通过后由后端借阅接口赋权并自动记录审计日志。

### 3. 保证金管理
*   **费用与保证金看板**：展示已支付总额、未退还总额、未退还笔数、待退还提醒卡片与列表明细。
*   **归还跟踪**：提供缴纳日期、应退日期、归还状态、收款人等字段清单。

## 三、 安全合规与系统集成约束

*   **数据访问权限隔离 (按角色细化控制)**：
    *   *投标专员*：在项目档案 Tab 下仅可查看/检索/导出**自己参与的项目**；在案例库 Tab 下拥有全量查看/复用权限，但无编辑、下架权限。
    *   *投标管理员/组长*：拥有全局的项目档案查看、台账导出、案例沉淀异步任务配置、案例编辑与下架权限。
    *   *行政人员*：在资质证书子模块中拥有增删改、上传/替换附件、接收到期提醒的全部管理权限。
*   **操作审计与日志追溯**：
    *   必须记录所有的预览、下载、台账导出行为，精确到操作人、时间、文件名与操作类型。
    *   导出台账和导出文件包均计入安全审计日志。

## 四、 现有系统 Gap 分析与开发架构决策

### 1. 重构与增量开发的决策
> [!IMPORTANT]
> **准确答复**：**直接针对现有代码进行增量迭代开发，严禁推翻重构！**

**决策依据**：
1.  **架构已高度重构规范化**：后端 `backend/src/main/java/com/xiyu/bid/` 下的 `casework` 与 `businessqualification` 包已经是高度解耦的微模块分层设计，遵守 **FP-Java Profile**：
    *   `domain/`：专门负责资质有效期、借还状态、提醒策略等纯核心业务决策。
    *   `application/`：负责增删改查、借用审批工作流、到期扫描等用例编排。
    *   `infrastructure/`：处理 JPA 实体与仓储。
    *   `qualification/`：作为兼容协议层。
2.  **OA 数据流集成成熟**：新 UI 的借还审批流已稳定接入 `workflowform` 的 `QUALIFICATION_BORROW` 流程，由 `ProjectAccessScopeService` 统一收口安全，贸然重构会引入极高的测试崩溃和权限漏洞风险。

### 2. 核心 Gap 对齐清单 (增量任务)
我们应当仅补齐现有代码中缺失或与 PRD 不一致的细节：

*   **前端增量优化点**：
    1.  **列表 Popover 悬浮明细**：在 `src/views/Knowledge/components/qualification/` 与项目档案列表中，给文件数添加鼠标 hover 悬停展示分类的 Popover 气泡。
    2.  **双轴日期选择器**：优化检索过滤条，支持上传区间和结项区间的独立多轴筛选。
    3.  **行点击交互行为**：实现除“操作”按钮列以外点击整行滑出 Drawer 的全局事件绑定。
*   **后端增量优化点**：
    1.  **文件包 ZIP 打包下载**：补充生成指定格式目录压缩包的服务，在 ZIP 包根目录包含导出的 `_台账.xlsx` 汇总表格，并将导出动作计入操作日志表中。
    2.  **AI 案例结项异步提取**：在 `casework` 模块下实现当项目状态进入 `CLOSED` 时的异步处理机，通过 API 读取标书与评分项进行切片后落库。
