# DD-BP-01《调研流程清单》

| 项目 | 内容 |
|---|---|
| 文档编号 | DD-BP-01 |
| 文档名称 | 《调研流程清单》 |
| 项目名称 | 西域数智化投标管理平台 |
| 文档用途 | 需求适配输入；访谈范围确认 |
| 适用阶段 | 蓝图设计及系统配置阶段 |
| 调研对象 | 客户关键用户、业务负责人、IT/集成负责人、管理层代表 |
| 编制角色 | 投标管理专家 |
| 当前版本 | V0.1 调研准备稿 |
| 日期 | 2026-04-27 |

## 1. 调研目标

本次关键用户调研的目标，是把客户在附件4《需求任务书》和附件6《需求功能清单》中提出的要求，拆解为可确认、可实施、可验收的业务流程、角色权限、数据口径、集成边界和上线证据。

调研不以“介绍系统功能”为主，而以“确认客户真实投标管理方式”为主。所有访谈结论后续将进入《投标管理系统项目需求适配分析报告》《投标管理系统项目配套需求适配确认书》《投标管理系统流程设计文档》《投标管理系统接口设计文档》和 UAT 用例。

## 2. 调研输入基线

## 10. 风险提示

| 风险 | 影响 | 建议处理 |
|---|---|---|
| 关键用户不到场 | 流程确认不完整，后续返工 | 每个一级域指定业务owner和替补人 |
| 附件6功能未裁剪 | UAT范围膨胀，验收口径不清 | 调研后立即形成附件6功能范围裁剪确认表 |
| 集成资料不足 | CRM/OA/企微联调延期 | 首场会议锁定接口人、测试环境、字段样例 |
| 报表口径不明 | 管理驾驶舱难验收 | 要求客户提供6张报表定义表或样表 |
| 权限例外未确认 | 上线后出现数据不可见或越权争议 | 访谈中单独确认跨部门、项目组、管理员例外规则 |
| 非功能口径冲突 | 性能、可用性、试运行验收争议 | 在蓝图确认件中固化最终指标和测试方法 |
