 人工录入标讯

标讯列表页顶部操作区提供【人工录入】按钮，投标专员、投标组长、投标管理员、项目负责人均可操作。
录入流程
人工录入采用分步填写：填写基本信息-填写项目评估表，复用标讯详情页布局，以创建模式打开：
操作人点击【人工录入】按钮，打开标讯详情页创建模式，此时为详情页标题为“新建标讯”，标题下方不展示标讯状态标签
仅展示【基本信息】Tab，所有字段为空待填写（见下方表单字段）
页面底部操作栏显示【保存】按钮，必填项未填写完成时置灰
点击【保存】：
校验通过 → 可保存基本信息
校验不通过 → 页面高亮错误字段，给出对应提示，【保存】按钮置灰
基本信息保存后，标讯创建，此时详情页标题变更为“编辑标讯”，标题下方展示标讯为【待分配】状态

AI解析辅助录入
系统支持通过AI解析自动提取标讯信息，降低人工录入成本。
入口方式：
标讯文件信息：支持上传招标文件（PDF/Word），系统自动解析提取字段
标讯信息字段：支持粘贴招标公告文本，系统自动解析提取字段
AI解析流程：
通过对应字段，上传文件或粘贴文本
操作人点击【AI解析】或【粘贴识别】按钮
系统调用接口进行解析
解析完成后，自动回填到表单对应字段
操作人检查并补充完善后提交
权限：投标管理员、投标组长、项目负责人、投标专员均可使用。

投标项目
投标项目管理模块是「西域数智化投标管理平台」的核心模块，贯穿从项目立项到结项复盘的全生命周期。通过数字化手段实现投标业务的标准化、可视化与智能化管理。
流程概览
流程说明：
1、项目立项数据主要来自于【标讯中心】和【CRM系统】：
（1）标讯中心的标讯可通过三方平台推送或手工录入／导入。投标管理员／组长将其分配给项目负责人后，由负责人填写项目评估。评估通过后，进入项目立项。
（2）CRM系统触发投标时，会查询投标系统中是否存在对应标讯：如存在，则自动关联（项目详情页面有标讯字段关联）；如不存在，则手动录入标讯并关联，然后进入项目待立项阶段（项目评估信息自动带入）。
2、项目立项以后开始投标过程的管理，从项目立项开始依次流转六个阶段，直至项目结项。

备注：CRM系统、标讯、项目三处关于项目信息的表单字段，保持一致，以CRM为准。

智能识别：AI自动读取招标文件，识别评分项（如技术方案、商务报价、资质业绩等）、章节数量、关键约束条件
任务生成：按章节和评分维度自动生成任务，包含任务名称、详细描述、交付物要求
前置适配：生成的任务清单适配投标启动会流程，包含招标文件研读、资格审查材料准备、技术方案撰写、商务报价编制、业绩证明整理、实施方案撰写、售后服务承诺、保证金办理等标准环节
来源追溯：展示拆解来源（招标文件名称）、识别评分项数量、章节数、拆解任务总数
注意：如果本项目涉及保证金（项目立项中有“是否需要保证金”字段），则保证金办理任务是必须生成的，并且增加字段：保证金金额、缴纳方式、收款方名称、收款账号、预计退还时间。要求执行人必须填写，才能提交。
1-3 用户可执行的操作

注意事项：拆解结果由AI生成，仅供参考。系统会提示用户「请逐条核对任务名称、详情、交付物，并分配执行人」。所有字段（任务名称、详细描述、交付物、执行人）为必填项，未完成不可提交。
AI评分标准解析
2-1 核心功能
系统自动解析招标文件中的评分标准，提取评分维度、权重、资质要求、技术要点、商务条款和废标红线。同时联动知识库（资质库、人员库、品牌授权库、业绩库）进行智能比对，对资质和人员条件做风险识别，输出「已满足 / 需关注 / 不满足」三种状态标签。

2-2 实现效果
评分标准提取：自动识别招标文件中的所有评分项，按编号、维度、具体指标、权重展示，并计算总分
资质要求识别（联动知识库）：逐条列出招标文件中的资质要求，自动比对知识库数据，给出三种状态：
已满足 知识库中已找到匹配项，条件符合
需关注 条件临界（如证书即将到期、人数刚好满足下限），建议人工复核
不满足 知识库中未找到匹配项或条件不符合，存在废标风险
技术要点提取：提取招标文件中的技术指标、功能要求、兼容性要求等，标注「硬指标」「功能」「兼容性」「加分项」等类型
商务条款解析：提取付款方式、履约保证金、交付周期、质保期、违约责任、知识产权归属等关键商务条款
废标红线标记：识别招标文件中明确的废标条款（如未按规定装订、保证金逾期、业绩造假、授权缺失、报价超控制价等），高亮警示
按钮状态联动：解析完成后，按钮显示红点/数字徽标提示风险项数量，提醒用户关注
2-3 用户可执行的操作

重要说明：AI评分标准解析仅做风险识别与提示，不做量化评分。系统不对投标成功率做任何预测或打分。预警规则需在系统设置中配置，校验规则由业务提供固定表单，后续支持后台配置化管理。
AI智能案例推荐
3-1 核心功能
基于当前项目招标文件的评分项，AI从案例库中智能匹配相似度最高的历史项目应答片段。匹配逻辑覆盖技术方案、商务方案、实施服务、资质业绩等全维度评分项，帮助投标团队快速复用优质历史内容。

3-2 实现效果
智能匹配：按评分项维度（如技术方案·整体架构、数据备份、安全防护、售后服务、类似业绩等）分别匹配最相关的历史案例
匹配度评分：每条案例显示匹配度百分比（如95%），匹配度≥85%标记为「优质」
评分项筛选：支持按评分项下拉筛选，查看特定评分维度下的推荐案例
关键词过滤：支持输入关键词进一步过滤推荐结果
案例信息展示：每张案例卡片展示：案例标题、应答原文摘要、来源项目、中标结果、历史复用次数
原文高亮阅读：案例详情页左侧展示案例元信息，右侧展示来源标书的完整原文，命中片段自动高亮标注
立项即推荐：项目刚立项、未上传招标文件时，也可基于项目基本信息触发初步推荐
3-3 用户可执行的操作

【数据来源】案例来源于历史投标项目的技术文件、商务文件、实施方案等文档，由系统自动归档入库。
【匹配逻辑】基于评分项语义相似度 + 项目类型匹配 + 关键词命中综合计算匹配度。
