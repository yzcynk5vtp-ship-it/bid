# Proposal: Implement Bidding Center (标讯中心)

## Summary
基于产品蓝图 V1.1 完整实现标讯中心模块，补齐 Tender 实体字段、列表列展示、详情页 Tab 结构、去重逻辑、操作日志、分步录入流程等核心缺口。

## Motivation
当前代码与蓝图 V1.1 标讯中心对比完成度仅 ~55%。主要差距：
- Tender 实体缺 12 个字段 (项目类型/联系人体系/人员分配显式字段等)
- 列表页仅展示 5 列 vs 蓝图 18 列
- 详情页缺少 3 Tab 布局 (特别是操作日志)
- 缺少去重逻辑
- 人工录入不是分步流程

## Approach
采用增量补齐策略，不破坏现有功能：
1. **后端先行**: Flyway 迁移 + Tender Entity 扩展 + 新增 Service/Policy
2. **前端跟进**: 列表列扩展 + 详情页 Tab 重构 + 分步录入
3. **验证收尾**: 架构测试 + E2E 测试 + 代码评审

## Risks
- 列数增加可能导致列表性能下降 → 后端分页 + 前端虚拟滚动
- 分步录入影响现有 ManualTenderDialog 使用 → 保留旧组件作为兼容入口
