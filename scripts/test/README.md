# 测试基线脚本目录

> 一旦我所属的文件夹有所变化，请更新我。

这里放 Playwright 与 API 联调测试的环境控制脚本。  
这些脚本不承载业务逻辑，只负责把本地 E2E 基线拉起、探活和关闭。

| 文件 | 地位 | 功能 |
|------|------|------|
| `start-api-e2e-stack.sh` | 启动脚本 | 准备 API 联调所需的 mock-AI 后端、前端预览和依赖容器；只复用带本地 marker/pid 的 Playwright 托管栈，避免跨 worktree 端口污染；清理端口后必须确认监听端口已释放，若仍有非托管后端健康响应会直接失败 |
| `stop-api-e2e-stack.sh` | 停止脚本 | 仅关闭由 Playwright 基线启动脚本拉起的本地联调环境 |
