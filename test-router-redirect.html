Router Redirect Test - Manual Verification

Test Purpose

Verify that 401 errors trigger proper Vue Router navigation instead of window.location.href

Pre-conditions

  • Development server running on http://localhost:1314
  • Mock data loaded
  • Router guards configured

Test Steps

Step 1: Open DevTools Console and Network tabs
Step 2: Navigate to http://localhost:1314/dashboard
Step 3: Log in if not already logged in
Step 4: In DevTools Console, execute: 
// Simulate token expiration
sessionStorage.setItem('token', 'expired-token')
sessionStorage.setItem('refreshToken', 'expired-refresh')
Step 5: Trigger an API call by clicking any menu item or refreshing the page
Step 6: Observe the behavior:
  • Should see error message: "登录已过期，请重新登录"
  • Should be redirected to /login
  • In Console, should see NO errors about navigation
  • URL should change to http://localhost:1314/login
Step 7: Check in Console that router.push was used (not window.location.href): 
// Should not see this pattern:
// "NavigationDuplicated" errors are OK and expected
Step 8: Verify session was cleared: 
console.log('Token:', sessionStorage.getItem('token'))
console.log('Refresh:', sessionStorage.getItem('refreshToken'))
// Both should be null

Expected Results

  • ✓ Redirect uses Vue Router (router.push)
  • ✓ Router guards are triggered during redirect
  • ✓ No console errors (except NavigationDuplicated which is handled)
  • ✓ Session state is properly cleared
  • ✓ No redirect loops occur

What Changed

Before:

// src/api/client.js (lines 106-108)
if (window.location.pathname !== '/login') {
  window.location.href = '/login'  // ❌ Bypasses router guards
}

After:

// src/api/client.js (lines 107-115)
// Use Vue Router for navigation to ensure guards are triggered
if (router.currentRoute.value.path !== '/login') {
  router.push('/login').catch((navError) => {
    // Ignore navigation aborted errors (e.g., user navigated away)
    if (navError.name !== 'NavigationDuplicated') {
      console.error('Navigation to login failed:', navError)
    }
  })  // ✓ Uses router, guards work correctly
}
